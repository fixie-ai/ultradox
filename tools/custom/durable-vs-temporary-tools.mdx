Custom tools in Ultravox come in two varieties: **durable** and **temporary**. Understanding when to use each type is crucial for effective development and production deployment.

## Quick Comparison

| Aspect | Temporary Tools | Durable Tools |
|--------|----------------|---------------|
| **Creation** | In call request body | Via [Tools API](/api-reference/tools/tools-post) |
| **Persistence** | Call-scoped only | Permanently stored |
| **Reusability** | Single call | Across calls and agents |
| **Iteration Speed** | Very fast | Requires API calls |
| **Production Use** | Development only | Recommended |
| **Web App Agents** | Not supported | Full support |
| **Team Collaboration** | Limited | Excellent |

## Temporary Tools

Temporary tools are defined inline when creating a call and exist only for that specific call session.

### When to Use Temporary Tools

✅ **Early Development**: Rapid prototyping and experimentation  
✅ **Testing New Ideas**: Quick iteration without API overhead  
✅ **One-off Use Cases**: Tools needed for a single specific call  
✅ **Development Environment**: Testing before creating durable versions  

### Creating Temporary Tools

```js
// Temporary tool defined in call creation
{
  "systemPrompt": "You are a helpful assistant...",
  "selectedTools": [
    {
      "temporaryTool": {
        "modelToolName": "sendNotification",
        "description": "Send a notification to the user",
        "dynamicParameters": [
          {
            "name": "message",
            "location": "PARAMETER_LOCATION_BODY",
            "schema": {
              "type": "string",
              "description": "Notification message to send"
            },
            "required": true
          }
        ],
        "http": {
          "baseUrlPattern": "https://api.example.com/notify",
          "httpMethod": "POST"
        }
      }
    }
  ]
}
```

### Temporary Tool Advantages

✅ **Fast Iteration**: No need to create/update/delete tools during development  
✅ **Immediate Testing**: Test tool changes with every new call  
✅ **No API Overhead**: Define tools directly in your application code  
✅ **Version Control**: Tool definitions live in your application codebase  

### Temporary Tool Limitations

❌ **Not Reusable**: Must redefine for every call  
❌ **No Web App Support**: Can't be used with agents created in Ultravox web app  
❌ **API Creation Only**: Only work with agents created via API  
❌ **No Team Sharing**: Can't share tools across team members easily  

## Durable Tools

Durable tools are created once via the Tools API and can be reused across multiple calls and agents.

### When to Use Durable Tools

✅ **Production Applications**: Stable, tested functionality  
✅ **Web App Agents**: Required for agents created in Ultravox web app  
✅ **Team Collaboration**: Share tools across team members  
✅ **Reusable Functionality**: Use same tool across multiple agents  
✅ **Stable APIs**: When tool definitions won't change frequently  

### Creating Durable Tools

```bash
# Create a durable tool via API
curl -X POST "https://api.ultravox.ai/api/tools" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{
    "name": "sendNotification",
    "definition": {
      "modelToolName": "sendNotification",
      "description": "Send a notification to the user",
      "dynamicParameters": [
        {
          "name": "message",
          "location": "PARAMETER_LOCATION_BODY",
          "schema": {
            "type": "string",
            "description": "Notification message to send"
          },
          "required": true
        }
      ],
      "http": {
        "baseUrlPattern": "https://api.example.com/notify",
        "httpMethod": "POST"
      }
    }
  }'
```

### Using Durable Tools

```js
// Reference durable tool by name or ID
{
  "systemPrompt": "You are a helpful assistant...",
  "selectedTools": [
    { "toolName": "sendNotification" }
    // or
    // { "toolId": "tool-uuid-here" }
  ]
}
```

### Durable Tool Advantages

✅ **Reusable**: Use across multiple calls and agents  
✅ **Web App Compatible**: Works with agents created in Ultravox web app  
✅ **Team Sharing**: Visible to all team members  
✅ **Version Control**: Update once, affects all usage  
✅ **Organized**: Manage tools separately from application logic  

### Durable Tool Limitations

❌ **Slower Iteration**: Requires API calls to create/update  
❌ **API Dependency**: Need to manage tool lifecycle via API  
❌ **Update Overhead**: Changes affect all existing usage  

## Development Workflow

### Recommended Development Path

<Steps>
  <Step title="Prototype with Temporary Tools">
    Start with temporary tools for rapid iteration and testing
    ```js
    // Quick prototype in call creation
    {
      "selectedTools": [
        { "temporaryTool": { /* your tool definition */ } }
      ]
    }
    ```
  </Step>
  
  <Step title="Stabilize and Test">
    Refine your tool definition through multiple iterations
    ```js
    // Update tool definition and test with new calls
    {
      "temporaryTool": {
        "modelToolName": "improvedTool",
        "description": "Updated description...",
        // refined parameters and implementation
      }
    }
    ```
  </Step>
  
  <Step title="Create Durable Version">
    Once stable, create a durable tool for production use
    ```bash
    # Create production-ready durable tool
    curl -X POST "https://api.ultravox.ai/api/tools" \
      -H "X-API-Key: your-api-key" \
      -d '{ "name": "finalTool", "definition": { /* stable definition */ } }'
    ```
  </Step>
  
  <Step title="Deploy and Reuse">
    Use the durable tool across all agents and applications
    ```js
    {
      "selectedTools": [
        { "toolName": "finalTool" }
      ]
    }
    ```
  </Step>
</Steps>

## Web App Integration

### Agents Created in Web App
- **✅ Durable Tools**: Can be selected and used
- **❌ Temporary Tools**: Not supported

### Agents Created via API
- **✅ Durable Tools**: Can be referenced by name or ID
- **✅ Temporary Tools**: Can be defined inline

```js
// API-created agent with both tool types
{
  "name": "Hybrid Agent",
  "callTemplate": {
    "selectedTools": [
      { "toolName": "durableTool" },        // Durable tool
      { "temporaryTool": { /* definition */ } } // Temporary tool
    ]
  }
}
```

<Warning>
**Web App Compatibility**

If you plan to use agents in the Ultravox web app or share them with team members who use the web app, you must use durable tools. Temporary tools are only available for agents created via API.
</Warning>

## Team Collaboration Scenarios

### Small Team / Solo Development
```js
// Temporary tools work well for small teams
{
  "selectedTools": [
    { "temporaryTool": { /* definition in codebase */ } }
  ]
}
```

### Large Team / Production
```bash
# Durable tools enable better collaboration
# Tool team creates tools
curl -X POST "https://api.ultravox.ai/api/tools" -d '{ /* tool definition */ }'

# Application teams use tools
{
  "selectedTools": [
    { "toolName": "teamSharedTool" }
  ]
}
```

## Best Practices

### For Temporary Tools
1. **Document Well**: Since they're not centrally managed, ensure good documentation
2. **Version in Code**: Keep tool definitions in your version control system
3. **Test Thoroughly**: Each call gets a fresh instance, so test edge cases
4. **Graduate to Durable**: Move stable tools to durable when ready

### For Durable Tools
1. **Semantic Versioning**: Use clear naming conventions (e.g., `toolName_v2`)
2. **Deprecation Strategy**: Plan how to retire old tool versions
3. **Team Communication**: Coordinate changes that affect multiple users
4. **Testing Environment**: Test changes before updating production tools

### Migration Strategy

When moving from temporary to durable tools:

1. **Create Durable Version**:
   ```bash
   curl -X POST "https://api.ultravox.ai/api/tools" \
     -d '{ "name": "myTool_v1", "definition": { /* from temporary */ } }'
   ```

2. **Update Call Creation**:
   ```js
   // Before (temporary)
   { "temporaryTool": { /* definition */ } }
   
   // After (durable)
   { "toolName": "myTool_v1" }
   ```

3. **Test and Validate**: Ensure the durable version works identically

4. **Deploy**: Update your application to use the durable tool

Choose the approach that fits your development stage and team structure. Start with temporary tools for rapid development, then graduate to durable tools for production stability and team collaboration.
